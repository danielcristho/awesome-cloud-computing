# Technology Stack

This is a documentation-focused project built around Markdown content curation and community contribution workflows.

## Tech Stack
- **Content Format**: Markdown (.md files)
- **Version Control**: Git with GitHub hosting
- **License**: MIT License
- **Documentation Standard**: Awesome List format with structured tables

## Build System
No build system required - this is a static documentation project that renders directly on GitHub.

## Content Structure
- Structured Markdown with consistent table formatting
- Awesome List badge compliance
- Organized sections with Table of Contents
- Standardized link format: `[Name](URL) - Description.`

## Quality Standards
- All links must be functional and lead to legitimate resources
- Descriptions should be concise and end with a period
- Resources should be categorized appropriately
- Maintain consistent formatting across all sections

## Common Commands
Since this is a documentation project, common operations are:

```bash
# Validate markdown formatting
markdownlint README.md

# Check for broken links (if using link checker)
markdown-link-check README.md

# Preview changes locally
# (Use any markdown preview tool or GitHub's preview)
```

## Contribution Workflow
- Fork repository
- Create feature branch
- Add single resource per PR
- Follow PR template guidelines
- Ensure proper categorization and formatting
