# Contributing Guidelines

Your contributions are always welcome! This guide will help you contribute effectively to the Awesome Cloud Computing list.

## How to Contribute

### Adding New Resources

1. **Fork** the repository
2. **Create** a new branch for your contribution
3. **Add** your resource following the guidelines below
4. **Submit** a pull request with a clear description

### Contribution Rules

- **One resource per Pull Request** - This makes it easier to review and discuss each addition
- **Use the established format** - Follow the existing table structure and formatting
- **Provide accurate information** - Ensure all links work and descriptions are correct
- **Keep descriptions concise** - End descriptions with a period for consistency
- **Categorize appropriately** - Place resources in the most relevant section

## Quality Standards

### Link Requirements

- All links must be **functional** and lead to legitimate resources
- Prefer **official documentation** or primary sources
- Avoid **affiliate links** or promotional content
- Use **HTTPS** when available

### Description Guidelines

- Keep descriptions **concise** but informative (1-2 sentences max)
- Focus on **what the tool/resource does** rather than marketing language
- End descriptions with a **period** for consistency
- Use **present tense** and active voice

### Formatting Standards

- Follow the existing **table format** with Name, Description, Link columns
- Use **proper markdown** syntax
- Maintain **alphabetical ordering** within sections where logical
- Use **consistent capitalization** for names and titles

## Pull Request Template

When submitting a PR, please use this format:

```
## Type of Change
- [ ] New resource
- [ ] Update existing resource
- [ ] Fix broken link
- [ ] Documentation improvement
- [ ] Other (please describe)

## Description
Brief description of what you're adding/changing and why.

## Checklist
- [ ] I have read the contributing guidelines
- [ ] The link is functional and leads to the correct resource
- [ ] The description is concise and ends with a period
- [ ] The resource is placed in the appropriate category
- [ ] I have tested the formatting in markdown
```

## What Not to Include

- **Broken or dead links**
- **Duplicate resources** (check existing entries first)
- **Promotional or marketing content**
- **Resources behind paywalls** (unless clearly marked)
- **Personal projects** without significant community adoption
- **Outdated or deprecated tools**

## Review Process

1. **Automated checks** - Links and formatting are validated
2. **Community review** - Other contributors may provide feedback
3. **Maintainer approval** - Final review by project maintainers
4. **Merge** - Approved contributions are merged into the main branch

## Suggestions for Contributions

We're always looking for:

- **New emerging technologies** in cloud computing
- **Updated links** to replace outdated ones
- **Better descriptions** for existing resources
- **New categories** for organizing content
- **Translations** to other languages

## Code of Conduct

Please note that this project follows our [Code of Conduct](https://github.com/danielcristho/awesome-cloud-computing/blob/main/CODE_OF_CONDUCT.md). By participating, you agree to abide by its terms.

## Questions?

If you have questions about contributing, please:

- Open an **issue** for discussion
- Check existing **issues** and **pull requests**
- Contact the maintainers directly

Thank you for helping make this resource better for the cloud computing community!
